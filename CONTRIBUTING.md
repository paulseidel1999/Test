# Contributing

## Development

1. Create a feature branch from `main`.
2. Implement focused changes.
3. Run:

```bash
npm test
```

## Pull Requests

Each PR should include:

- Summary of changes
- Why the change is needed
- Testing evidence

## Review Flow

1. QA validates behavior and test results.
2. Product/owner approves scope and merge.
3. Merge to `main` after QA + approval.

## If Blocked

If a task cannot proceed, add a PR or ticket comment with:

- What is blocked
- Who needs to unblock it
- The exact decision or access needed
